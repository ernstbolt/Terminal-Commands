# Locatie van demo scripts

## Standaard structuur

Alle demo scripts worden opgeslagen binnen de **Lessen map**, per les:

```text
/mnt/c/Users/abc/Lessen/
```

Elke les heeft een eigen map, bijvoorbeeld:

```text
/mnt/c/Users/abc//Lessen/Les-week-11-Kubernetes-Orchestratie
```

---

## Bestanden per les

Binnen een lesmap staan doorgaans:

```text
demo-week-11.sh     → bash script (launcher)
kube-demo.md        → markdown met commands
```

👉 Belangrijk:

- script en markdown staan in **dezelfde map**
    
- script verwijst naar markdown bestand
    

---

## Voordelen van deze structuur

- logische indeling per les
    
- eenvoudig terug te vinden
    
- makkelijk te versiebeheren (Git)
    
- demo’s zijn herbruikbaar
    

---

# Bash script gebruik

Het script:

- leest het markdown bestand
    
- toont een lijst met stappen
    
- laat een preview zien
    
- voert het gekozen command direct uit
    

Starten kan via:

```bash
./demo-week-11.sh
```
